## 1. structuredClone

원본에 영향을 주지 않고 객체 속성을 추가 해야 할 때, `structuredClone` 함수를 사용해 복사 할 수 있습니다.
객체 복사는 spread operator를 사용하면 되는거 아니냐구요? 물론 가능하지만 다음과 같은 문제가 발생 할 수 있습니다.

```Javascript
const fruits = {
  types: {
    isFresh: true
  }
}

const copyFruits = { ...fruits }

copyFruits.name = 'Apple'
copyFruits.types.size = 'large'

console.log(fruits.name) // undefined
console.log(fruits.types.size) // 'large'
```
위의 코드와 같이 복사한 `copyFruits` 변수에 `types.size` 속성을 추가했을 뿐 인데 원본인 `fruits` 변수에도 `types.size` 속성이 생겨버렸습니다.
이처럼 spread operator와 같은 얕은복사는 중첩된 속성값이 추가되는 경우 의도와 다르게 원본이 같이 변경 될 수 습니다. 이럴 경우 `structuredClone` 함수를 통해 깊은복사 하여 해결 할 수 있습니다.

```Javascript
const fruits = {
  types: {
    isFresh: true
  }
}

const copyFruits = structuredClone(fruits)

copyFruits.name = 'Apple'
copyFruits.types.size = 'large'

console.log(fruits.name) // undefined
console.log(fruits.types.size) // undefined
```
